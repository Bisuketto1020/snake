貪吃蛇遊戲規格書（依現行程式碼）
1. 文件目的
本文件描述目前專案已實作之貪吃蛇遊戲功能、操作方式、畫面規格、遊戲規則與已知限制，作為後續維護與擴充依據。

2. 專案範圍
平台：桌面瀏覽器（以鍵盤操作為主）。
架構：純前端（HTML/CSS/JavaScript），無後端、無資料庫。
執行方式：開啟 index.html 即可執行。
3. 畫面與介面規格
3.1 主要區塊
遊戲棋盤：30 x 30 格，共 900 格。
控制區：
Start 按鈕：開始新遊戲（會重置目前局）。
Pause 按鈕：暫停 / 繼續。
Score：顯示目前分數。
3.2 視覺樣式
整體背景：黑色。
棋盤背景：深灰色。
蛇身：白色。
食物：紅色。
棋盤尺寸：600px x 600px。
版面置中，控制列位於棋盤下方。
4. 核心遊戲規則
4.1 初始化
頁面載入時先建立棋盤（30x30）。
按 Start 後：
蛇初始座標為 (2,0)、(1,0)、(0,0)，朝右移動。
分數重設為 0。
速度重設為 200ms / tick。
隨機產生 1 個食物。
4.2 移動與方向控制
以鍵盤方向鍵控制（↑ ↓ ← →）。
每個遊戲 tick 依當前方向前進一格。
禁止直接 180 度反向（例如向右時不可立刻向左）。
4.3 邊界行為
採「穿牆」機制：
超出左側會從右側出現。
超出右側會從左側出現。
超出上側會從下側出現。
超出下側會從上側出現。
4.4 吃到食物
當蛇頭與食物座標重疊：
分數 +1。
蛇身成長（該回合不刪尾）。
立即隨機產生新食物。
遊戲速度加快：每次吃到食物，間隔 -2ms。
最快上限為 10ms / tick。
4.5 失敗條件
蛇頭移動到任一蛇身座標時判定 Game Over。
發生 Game Over 後：
停止遊戲迴圈。
顯示瀏覽器 alert("Game Over")。
5. 遊戲流程（狀態觀點）
Idle（待機）：
頁面初始狀態，棋盤已建立但遊戲未開始。
Running（進行中）：
Start 後進入，定時更新位置與重繪。
Paused（暫停）：
Running 時按 Pause 進入，再按 Pause 回到 Running。
GameOver（結束）：
發生自撞後進入；需按 Start 才能重新開始。
6. 資料與邏輯規格
6.1 主要資料
snake：陣列，儲存蛇身每節座標（索引 0 為蛇頭）。
direction：目前移動方向向量。
nextDirection：下一次 tick 生效方向（避免同 tick 多次鍵入造成混亂）。
food：食物座標。
score：分數。
speed：遊戲間隔（ms）。
gameInterval：setInterval 計時器識別。
isRunning：是否可更新遊戲狀態。
6.2 每個 tick 的處理順序
套用 nextDirection。
計算新蛇頭座標並執行穿牆換位。
檢查是否撞到蛇身（是則結束）。
蛇頭加入陣列前端。
檢查是否吃到食物：
吃到：加分、加速、重設計時器、保留尾巴（成長）。
未吃到：刪除尾巴。
清空舊樣式並重繪食物與蛇身。
7. 事件與互動規格
Start 按鈕 click：觸發新局初始化與啟動。
Pause 按鈕 click：若正在遊戲則切換暫停/繼續。
window keydown：接收方向鍵輸入並更新 nextDirection。
8. 已知限制與現況說明
食物隨機位置目前未避開蛇身，可能生成在蛇體上。
暫停時 isRunning 仍為 true，但以清除計時器方式停止更新（功能可用）。
無觸控控制、無難度選單、無音效、無最高分保存。
Game Over 使用原生 alert，無客製化對話框。
9. 驗收條件（依現行功能）
可正常開始、暫停、繼續、重啟遊戲。
方向鍵可控制蛇移動，且不能直接反向。
邊界穿牆正確。
吃到食物可得分、長度增加且速度提升。
自撞時遊戲停止並顯示 Game Over。
分數顯示會即時更新。
10. 後續擴充建議（非本次範圍）
食物避開蛇身座標。
新增最高分（localStorage）。
手機觸控方向鍵。
分級速度或地圖障礙模式。
Game Over 視覺化面板與重新開始按鈕。
